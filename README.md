# Brazilian E-Commerce Sales and Delivery Analytics

## Project Overview

This project analyzes the Olist Brazilian E-Commerce Dataset using Python and Tableau. The dataset captures the full e-commerce transaction flow, including customers, orders, order items, payments, reviews, products, sellers, geolocation, and product category translations.

The objective is to turn the raw CSV files into analysis-ready tables that can be used for business reporting, data storytelling, and dashboard development. The notebook focuses on practical analytics questions around revenue, sales volume, customer satisfaction, delivery performance, and regional trends.

## Why This Project Matters

This dataset is useful because it connects multiple business functions in one place:

- Commercial performance through revenue and sales volume
- Customer behavior through payment patterns and review scores
- Operational performance through delivery speed and late deliveries
- Geographic performance through state and city-level analysis
- Product performance through category trends and item-level sales

That combination makes it a strong end-to-end analytics project for both Python exploration and Tableau visualization.

## Notebook Workflow

The main notebook, `DataAnalytics1.ipynb`, follows this workflow:

1. Import Python libraries.
2. Load all raw CSV datasets.
3. Inspect dataset size, missing values, duplicate rows, and summary statistics.
4. Clean and standardize the data.
5. Convert timestamps to datetime values.
6. Create new fields such as delivery days, delivery delay, and late-delivery flags.
7. Merge tables into order-level and item-level analysis datasets.
8. Group and aggregate data for revenue, sales, ratings, and delivery metrics.
9. Analyze monthly trends and category growth.
10. Build charts for reporting and dashboard use.
11. Export cleaned, Tableau-ready CSV files.

## Data Cleaning and Transformation

The notebook performs the following transformations:

- Removes duplicate rows from the major tables.
- Fills missing product category names with `Unknown`.
- Replaces missing review comments with `No Review`.
- Converts date columns into datetime format.
- Computes delivery duration in days.
- Computes delivery delay relative to the estimated delivery date.
- Flags orders that were delivered late.
- Translates product category names into English where possible.

These steps make the data easier to analyze and reduce errors in downstream metrics.

## Key Analysis Tables

The notebook creates analysis-ready datasets:

- `order_base`: one row per order with customer, payment, delivery, and review information.
- `item_base`: one row per order item with product, seller, category, and review information.
- `tableau_full`: one row per order item, enriched with all major analytical fields for Tableau.

These tables are useful because they separate order-level analysis from item-level analysis, which helps avoid double counting in revenue metrics.
